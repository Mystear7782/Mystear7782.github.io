## アサーション方針（v1.1変更点）

ページ遷移の確認は `to_have_class("active")` を使用しない。アプリの各ページ要素は `class="page active"` と複数クラスを持つため、`to_have_class` は完全一致で失敗する。

```python
# NG: class="page active" に対して失敗する
expect(page.locator("#page-set-edit")).to_have_class("active")

# OK: CSSセレクタで .active を含む要素の存在を確認する
expect(page.locator("#page-set-edit.active")).to_be_visible()
# または wait_for_selector で待機してから確認
page.wait_for_selector("#page-set-edit.active")
```

```

TC-01-11: 半角61文字を fill() すると全角換算30文字でカット（実測値30文字）

```

## 機能追加時のテストメンテナンス方針

新機能を追加した際は以下の手順でテストをメンテナンスする。

1. **UCの追加・更新**: `usecase.md` にユースケースを追記する
2. **TCの追加**: 本ドキュメントのTC一覧に行を追加する（状態は「未実装」で登録）
3. **テストコードの追加**: 対応するテストファイルにテスト関数を追加する
4. **既存テストへの影響確認**: 変更した機能に関連する既存TCの期待値・手順を見直す
5. **全件実行**: `python -m pytest tests/e2e -v --html=reports/report.html --self-contained-html`
6. **ドキュメント更新**: 全件Passしたら本ドキュメントの状態列を `✅ Pass` に更新する
