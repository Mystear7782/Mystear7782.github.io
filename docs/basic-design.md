## 1. プロジェクト概要

### 1.1 目的

個人の筋トレトレーニングを記録・管理するためのWebアプリケーション。スマートフォン（iPhone）のホーム画面からネイティブアプリと同等の操作感で利用できることを目指す。

### 1.2 開発方針

- PWA（Progressive Web App）方式で実装し、App Store申請・Macなしで開発・公開できる構成とする
- データはiPhone本体のlocalStorageに保存し、サーバーへの通信なしで動作する
- GitHub Pagesを利用して無料でホスティングする

### 1.3 対象ユーザー

個人利用（開発者自身）

---

## 2. システム構成

### 2.1 技術スタック

| 項目 | 採用技術 | 理由 |
|---|---|---|
| フロントエンド | HTML / CSS / JavaScript（バニラ） | 追加ツール不要・即時デプロイ可能 |
| フォント | Noto Sans JP（Google Fonts） | 日本語表示の視認性 |
| ホスティング | GitHub Pages（無料） | Publicリポジトリで無料利用可能 |
| データ保存 | localStorage | オフライン動作・サーバー不要 |
| オフライン対応 | Service Worker | PWAとして動作させるために必要 |
| グラフ描画 | Chart.js 4.4.1（CDN遅延読み込み） | 分析詳細画面のグラフ表示 |

### 2.2 システム構成図

```
開発環境（Windows PC）
　└── VS Code でコード編集
　└── Git でバージョン管理

GitHub（Publicリポジトリ）
　└── main ブランチに push → GitHub Pages に自動反映

iPhone（Safari）
　└── GitHub Pages の URL にアクセス
　└── 「ホーム画面に追加」でPWAとしてインストール
　└── localStorage にデータ保存（端末内）
```

### 2.3 ファイル構成

```
gym-log/
├── index.html            # アプリ本体（HTML / CSS / JS を1ファイルに集約）
├── manifest.json         # PWA設定（アプリ名・アイコン・表示モード）
├── sw.js                 # Service Worker（オフライン対応・キャッシュ制御）
├── apple-touch-icon.png  # iPhoneホーム画面アイコン（180×180）
├── icon-192.png          # PWAアイコン（192×192）
└── icon-512.png          # PWAアイコン（512×512）
```

## 5. 非機能要件

### 5.1 動作環境

| 項目 | 要件 |
|---|---|
| 対象端末 | iPhone（Safari / PWAモード） |
| オフライン動作 | 必須（ジムでの利用を想定） |
| ダークモード | 端末設定に自動連動（`prefers-color-scheme`） |
| フォント | Noto Sans JP |

### 5.2 セキュリティ

| 項目 | 方針 |
|---|---|
| データ保存先 | iPhone本体のlocalStorage（サーバー保存なし） |
| GitHub リポジトリ | Public（コードは公開されるがデータは非公開） |
| APIキー等の機密情報 | コードへの記載禁止 |
| HTTPS | GitHub Pagesにより自動有効 |
| 検索エンジン対策 | robots.txtで必要に応じてクロール制限（将来対応） |

### 5.3 データ保存

| 項目 | 内容 |
|---|---|
| 保存方式 | localStorage（キー: `gl_menus`, `gl_sessions`） |
| 消失リスク | Safariのデータ消去操作・機種変更時・PWAアイコン削除時にデータ消失の可能性あり |
| バックアップ手段 | CSVエクスポート機能（実装済み） |
| データ移行 | 旧形式（`gl_history`）から新形式（`gl_sessions`）への自動マイグレーション実装済み |

### 5.4 注意事項（PWAアイコン更新時）

iPhoneのホーム画面からPWAアイコンを削除するとlocalStorageも消去される。アイコン更新時は必ずCSVエクスポートでバックアップを取ること。安全な更新手順は以下の通り。

```
1. CSVエクスポートでデータをバックアップ
2. ホーム画面のアイコンを削除せず、SafariでURLを直接開く
3. 「ホーム画面に追加」で新しいアイコンとして再追加
4. 旧アイコンを削除（この時点では新アイコン側にデータがある）
```

## 6. デプロイ・運用

### 6.1 デプロイ手順

```
1. main ブランチに変更を push
2. GitHub Pages に自動反映（数分以内）
3. iPhoneでアプリスイッチャーから完全終了 → 再起動で反映
```

### 6.2 Service Workerのバージョン管理方針

| 状況 | 対応 |
|---|---|
| index.html・CSSの変更 | バージョン変更不要（ネットワーク優先で自動更新） |
| sw.js自体のロジック変更 | `CACHE`定数のバージョンを上げる（例: v3 → v4） |
| 古いキャッシュによる不具合 | バージョンを上げてpush |
| 現在のバージョン | `gymlog-v3` |

### 6.3 PWA更新が反映されない場合の対処

1. アプリスイッチャーから完全終了 → 再起動
2. それでも反映されない場合：SafariでURLを直接開き「ホーム画面に追加」で再追加（データ消失に注意）

2. それでも反映されない場合：ホーム画面のアイコンを削除して再追加（Service Workerの完全リセット）
