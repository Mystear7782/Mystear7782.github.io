## 1. データ設計

### 1.1 localStorage キー一覧

| キー | 型 | 概要 |
|---|---|---|
| `gl_menus` | JSON配列 | 登録メニューの一覧 |
| `gl_sessions` | JSONオブジェクト | トレーニング記録（セッション単位） |

### 1.2 gl_menus スキーマ

```json
[
  {
    "id":       "menu_1700000000000",
    "name":     "ベンチプレス",
    "category": "胸",
    "type":     "フリーウェイト",
    "archived": false
  }
]
```

| フィールド | 型 | 説明 |
|---|---|---|
| id | string | `menu_` + `Date.now()` で生成 |
| name | string | メニュー名（全角30文字以内） |
| category | string | 部位（胸 / 背中 / 肩 / 足 / 腕 / 有酸素運動） |
| type | string | 種別（マシン / フリーウェイト / 自重運動 / 有酸素運動） |
| archived | boolean | アーカイブ状態（true: 一覧非表示） |

### 1.3 gl_sessions スキーマ

セッションは筋トレ・有酸素運動でデータ構造が異なる。カテゴリによって `sets` または `cardio` のどちらかが格納される。

**筋トレセッション**

```json
{
  "menu_1700000000000": {
    "sess_1700000001000": {
      "date": "2026-03-21",
      "sets": [
        { "w": 80, "r": 10 },
        { "w": 85, "r": 8 }
      ]
    }
  }
}
```

**有酸素運動セッション**

```json
{
  "menu_1700000000001": {
    "sess_1700000002000": {
      "date": "2026-03-21",
      "cardio": {
        "time":   30,
        "dist":   5.0,
        "cal":    300,
        "hr":     140,
        "maxSpd": 12.0,
        "avgSpd": 8.5
      }
    }
  }
}
```

| フィールド | 型 | 説明 |
|---|---|---|
| キー（menuId） | string | gl_menus の id と対応 |
| キー（sessionId） | string | `sess_` + `Date.now()` で生成 |
| date | string | 記録日（YYYY-MM-DD形式） |
| sets | 配列 | 筋トレのみ。セットの配列 `[{w, r}]` |
| sets[].w | number | 使用重量（kg） |
| sets[].r | number | 挙上回数（回） |
| cardio | オブジェクト | 有酸素運動のみ |
| cardio.time | number \| null | 合計所要時間（分）・必須 |
| cardio.dist | number \| null | 合計距離（km）・必須 |
| cardio.cal | number \| null | 合計消費カロリー（kcal）・必須 |
| cardio.hr | number \| null | 平均心拍数（bpm）・必須 |
| cardio.maxSpd | number \| null | 最大速度（km/h）・任意 |
| cardio.avgSpd | number \| null | 平均速度（km/h）・任意 |

### 1.4 マイグレーション

旧バージョンのデータ形式（`gl_history`）から新形式（`gl_sessions`）への変換を初回起動時に自動実行する。

```
旧: gl_history[menuId][date] = [{w, r}, ...]
新: gl_sessions[menuId][sessionId] = { date, sets:[{w, r}] }
```

変換後は `gl_history` キーを削除する。

## 2. ロジック設計

### 2.1 1RM計算式

O'Conner式を採用する。

```
1RM = 重量 × (1 + 回数 ÷ 40)
```

実装：

```javascript
const orm = (w, r) => +(w * (1 + r / 40)).toFixed(1);
```

小数点第1位で四捨五入して返す。

### 2.2 メニュー統計計算（menuStats）

メニュー詳細画面の統計値を算出する。カテゴリによって算出内容が異なる。

**筋トレの場合**

```
入力: menuId
処理:
  1. gl_sessions[menuId] の全セッションをループ
  2. 各セッションの全セットで1RMを計算し、セッション内最大値を取得
  3. 全セッション横断で最大1RM・最大ボリューム・最終日付を更新
出力: { maxOrm, maxVol, lastDate, isCardio:false }
```

**有酸素運動の場合**

```
入力: menuId
処理:
  1. gl_sessions[menuId] の全セッションをループ
  2. cardio.dist・cardio.cal の最大値と最終日付を更新
出力: { maxDist, maxCal, lastDate, isCardio:true }
```

### 2.3 メニュー名バリデーション

```
- 全角・半角を問わず文字数を [...value].length でカウント
- 30文字を超えた時点で入力を即時カット（超過分を破棄）
- サロゲートペア（絵文字等）も1文字としてカウント
```

### 2.4 日付フォーマット

```javascript
// 入力: "2026-03-21"
// 出力: "2026年3月21日"
const fmtDate = d => {
  const [y, m, day] = d.split('-');
  return `${y}年${parseInt(m)}月${parseInt(day)}日`;
};
```

## 3. 画面詳細設計

### 3.1 メニューの追加（add-menu）

#### 入力項目

| 項目 | 部品 | バリデーション |
|---|---|---|
| 部位 | セレクトボックス | 必須 |
| 種別 | チップ選択（複数候補から1つ） | 必須（有酸素運動選択時は非表示・自動セット） |
| メニュー名 | テキスト入力 | 必須・全角30文字以内 |

#### 追加ボタンの活性条件

部位・種別・メニュー名がすべて入力されている場合のみ活性化。

#### 重複チェック

同一部位に同名のメニュー（アーカイブ済みを除く）が存在する場合はトーストでエラー表示し、追加しない。

#### 有酸素運動の特殊処理

部位に「有酸素運動」を選択した場合、種別選択UIを非表示にし、`type` に「有酸素運動」を自動セットする。

---

### 3.2 メニュー一覧（menu-list）

#### 表示ロジック

```
1. gl_menus をフィルタリング
   - 通常表示: archived === false のみ
   - アーカイブ表示ON時: すべて表示
2. CATS 配列（['胸','背中','肩','足','腕','有酸素運動']）の順でグループ化
3. 各グループ内はgl_menus への登録順で表示
4. 各メニュー行に最大1RMと最終記録日を表示
```

#### アーカイブトグル

- 右上の「アーカイブを表示」ボタンで切替
- アーカイブ済みメニューは破線ボーダー・opacity 0.6 で区別
- アーカイブ済みバッジ（amber色）を表示

---

### 3.3 メニュー詳細（menu-detail）

#### 表示内容

- メニュー名・タグ（種別・部位・アーカイブ状態）
- 統計グリッド：最大1RM / 最大ボリューム / セッション数
- アクションボタン群
- セッション履歴テーブル（日付降順・同日は新しいセッションIDが上）

#### アクションボタン

| ボタン | 表示条件 | 動作 |
|---|---|---|
| 新規セッション | archived === false のみ | 新しいsessionIdを発行してセット記録画面へ |
| アーカイブ | archived === false のみ | 確認ダイアログ → archived を true に変更 |
| アーカイブ解除 | archived === true のみ | archived を false に変更 |
| 完全削除 | 常に表示 | 確認ダイアログ → menus・sessions から削除 |

#### セッション履歴テーブル列

| 列 | 内容 |
|---|---|
| 日付 | セッションの記録日 |
| セット数 | そのセッションのセット数 |
| 最大重量 | そのセッション内の最大使用重量 |
| ボリューム | 重量 × 回数 の合計（kg） |
| 1RM | そのセッション内の最大推定1RM |

### 3.4 セット記録（set-edit）

#### 新規セッション作成フロー

```
メニュー詳細の「新規セッション」押下
  └── sessionId = 'sess_' + Date.now() を発行
  └── gl_sessions[menuId][sessionId] = { date: today(), sets: [] } で初期化
  └── セット記録画面へ遷移
```

#### 既存セッション編集フロー

```
メニュー詳細の履歴行を押下
  └── 押下した行の sessionId を S.sessionId にセット
  └── セット記録画面へ遷移（既存データを表示）
```

#### 日付変更

- 画面上部のdateインプットから自由に変更可能
- 変更時はリアルタイムで `gl_sessions[menuId][sessionId].date` を更新・保存

#### セット追加

| 入力 | 部品 | inputmode |
|---|---|---|
| 重量（kg） | number input | decimal（小数点対応） |
| 回数 | number input | numeric（整数のみ） |

- 両方入力された時点で推定1RMをリアルタイムプレビュー表示
- 「セットを追加」ボタン押下でセット配列に追加・保存

#### セット編集

- セット行をタップするとフォームに値を展開し編集モードへ
- 編集中の行は accent カラーでハイライト
- 「変更を保存」押下で該当インデックスを更新
- キャンセルは入力フィールドをクリアしてフォームをリセット

#### セット削除

- 各セット行の ✕ ボタンで確認ダイアログ後に削除
- 編集中のセットを削除した場合はキャンセル処理を同時実行

---

### 3.5 有酸素運動セット記録（set-edit / cardio）

カテゴリが「有酸素運動」のメニューでセット記録画面を開いた場合、重量・回数フォームの代わりに有酸素専用フォームを表示する。

#### 表示切替ロジック

```
isCardioMenu(S.menu) === true
  → 有酸素専用フォームを表示
isCardioMenu(S.menu) === false
  → 既存の筋トレフォームを表示（変更なし）
```

#### 入力項目

| 項目 | フィールドID | 単位 | inputmode | 必須 |
|---|---|---|---|---|
| 合計時間 | c-time | 分 | numeric | ✅ |
| 合計距離 | c-dist | km | decimal | ✅ |
| 消費カロリー | c-cal | kcal | numeric | ✅ |
| 平均心拍数 | c-hr | bpm | numeric | ✅ |
| 最大速度 | c-maxspd | km/h | decimal | 任意 |
| 平均速度 | c-avgspd | km/h | decimal | 任意 |

#### 保存フロー

```
入力のたびに saveCardio() が呼ばれる
  └── cardio オブジェクトをリアルタイムで更新・persist()
  └── 画面下部のプレビューカードをリアルタイム更新

「記録を保存」押下 → saveCardioFinal()
  └── 必須4項目（time / dist / cal / hr）の未入力チェック
  └── 未入力があればトーストでフィールド名を通知 → 保存しない
  └── 全必須項目が入力済みであれば menu-detail へ遷移
```

#### プレビュー表示

入力済みの項目のみカード形式でリアルタイム表示する。null の項目は非表示。

#### 詳細画面との対応

有酸素運動メニューの詳細画面では以下に切り替わる。

| 要素 | 筋トレ | 有酸素運動 |
|---|---|---|
| 統計グリッド | 最大1RM / 最大ボリューム / セッション数 | 最大距離 / 最大消費Cal / セッション数 |
| 履歴テーブル列 | 日付 / セット数 / 最大重量 / ボリューム / 1RM | 日付 / 時間 / 距離 / 消費Cal / 平均心拍 |

## 5. Service Worker設計

### 5.1 キャッシュ戦略

| リソース種別 | 戦略 | 理由 |
|---|---|---|
| index.html / .js ファイル | ネットワーク優先 → キャッシュ fallback | 常に最新を表示するため |
| フォント等の外部リソース | キャッシュ優先 → ネットワーク fallback | オフライン対応・通信節約 |

### 5.2 インストール・アクティベート

```
install:
  - ASSETS（index.html, manifest.json）をキャッシュに追加
  - skipWaiting() で即座にアクティブ化

activate:
  - 現在のCACHE名以外の古いキャッシュを全削除
  - clients.claim() でページを即座に制御下に置く
```

### 5.3 現在のキャッシュバージョン

`gymlog-v3`

## 6. 状態管理設計

アプリ全体の状態を `S` オブジェクトで一元管理する。

```javascript
const S = {
  menus:        [],      // gl_menus の内容（配列）
  sessions:     {},      // gl_sessions の内容（オブジェクト）
  menu:         null,    // 現在選択中のメニューオブジェクト
  sessionId:    null,    // 現在編集中のセッションID
  editingSetIdx: null,   // 編集中のセットインデックス（null: 新規追加モード）
  showArchived: false,   // アーカイブ表示トグルの状態
};
```

データの変更は必ず `persist()` を呼び出してlocalStorageに永続化する。

```javascript
const persist = () => {
  localStorage.setItem('gl_menus',    JSON.stringify(S.menus));
  localStorage.setItem('gl_sessions', JSON.stringify(S.sessions));
};
```

### 7.2 インポート

#### 処理方針

| ケース | 動作 |
|---|---|
| 重複セッション（session_idが一致） | スキップ（既存データ優先・INSERT方式） |
| アプリ未登録のメニュー名 | 自動で新規メニューを登録してから取り込む |
| session_idが空欄 | 同日・同メニューの行を同一セッションに自動まとめ |
| session_idが記載あり | そのIDで重複チェック（GymLogからの再インポート向け） |

#### session_id の扱い

```
【GymLogからエクスポートしたCSVを再インポートする場合】
  session_idあり → 既存IDと照合 → 重複ならスキップ

【他アプリ・手入力CSVをインポートする場合】
  session_id空欄 → 同日同メニューをキーに自動セッションID発行
  例: 同じ日付・同じメニュー名の行は全て1セッションにまとめられる
```

手入力CSVのサンプル（session_id空欄）：

```csv
menu_name,category,type,date,session_id,set_no,weight_kg,reps
ベンチプレス,胸,フリーウェイト,2026-03-10,,1,80,10
ベンチプレス,胸,フリーウェイト,2026-03-10,,2,85,8
ベンチプレス,胸,フリーウェイト,2026-03-10,,3,82,6
```

#### 結果表示

インポート完了後、画面上に以下の件数を表示する。

- インポートした件数
- 未登録メニューの新規登録件数
- 重複スキップ件数
