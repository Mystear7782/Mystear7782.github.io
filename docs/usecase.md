## 詳細

### UC-01 メニューを追加する

**アクター：** ユーザー
**事前条件：** なし
**基本フロー：**
1. サイドバーから「メニューの追加」を開く
2. 部位を選択する
3. 種別を選択する（有酸素運動の場合はスキップ）
4. メニュー名を入力する（全角30文字以内）
5. 「追加する」ボタンを押す
6. トーストで完了通知が表示される

**代替フロー：**
- 同一部位に同名のメニューが存在する場合 → エラートーストを表示し追加しない
- 部位・種別・名前が未入力の場合 → 「追加する」ボタンが非活性

---

### UC-06 筋トレのセットを記録する

**アクター：** ユーザー
**事前条件：** 筋トレメニューが1件以上登録されている
**基本フロー：**
1. メニュー一覧からメニューを選択する
2. メニュー詳細の「新規セッション」ボタンを押す
3. セット記録画面で重量（kg）と回数を入力する
4. 推定1RMがリアルタイムで表示される
5. 「セットを追加」ボタンを押す
6. セットが一覧に追加される
7. 手順3〜6を繰り返す
8. 「詳細に戻る」で保存完了

**代替フロー：**
- セッションの日付を変更したい場合 → 日付入力フィールドから変更

---

### UC-11 有酸素運動を記録する

**アクター：** ユーザー
**事前条件：** 有酸素運動のメニューが1件以上登録されている
**基本フロー：**
1. メニュー一覧から有酸素運動メニューを選択する
2. メニュー詳細の「新規セッション」ボタンを押す
3. 専用フォームで必須項目（時間・距離・消費Cal・心拍数）を入力する
4. 任意項目（最大速度・平均速度）を入力する（省略可）
5. 「記録を保存」ボタンを押す
6. メニュー詳細に戻る

**代替フロー：**
- 必須項目が未入力の場合 → 未入力フィールド名をトーストで通知し保存しない

---

### UC-16 カレンダーでトレーニング日を確認する

**アクター：** ユーザー
**事前条件：** セッションが1件以上記録されている
**基本フロー：**
1. サイドバーから「メニュー分析」を開く
2. カレンダータブが表示される
3. 記録がある日付がアクセントカラーで強調表示される
4. 前月・翌月ボタンで月を移動できる

---

### UC-17 カレンダーの日付からセッション一覧を確認する

**アクター：** ユーザー
**事前条件：** 該当日に記録が存在する
**基本フロー：**
1. カレンダーの記録がある日付をタップする
2. カレンダー下部にその日のセッション一覧が展開される
3. 筋トレ：セット数・最大1RM・ボリュームが表示される
4. 有酸素運動：時間・距離・消費カロリーが表示される

---

### UC-18 カレンダーからセッション詳細に遷移する

**アクター：** ユーザー
**事前条件：** UC-17でセッション一覧が表示されている
**基本フロー：**
1. セッション一覧の行をタップする
2. 該当セッションのセット記録画面に遷移する
3. セットの確認・編集ができる
4. 「詳細に戻る」ボタンでメニュー分析画面に戻る

---

### UC-19 ボリューム推移グラフを確認する

**アクター：** ユーザー
**事前条件：** 筋トレメニューに1件以上のセッション記録がある
**基本フロー：**
1. メニュー分析の「メニュー」タブを開く
2. 対象メニューを選択する
3. 分析詳細画面でボリューム推移の折れ線グラフが表示される
4. X軸が日付、Y軸がボリューム（kg）で表示される

---

### UC-23 CSVをインポートして記録を取り込む

**アクター：** ユーザー
**事前条件：** インポート可能なCSVファイルが存在する
**基本フロー：**
1. サイドバーから「CSV出力 / 入力」を開く
2. 「CSVファイルを選択してインポート」ボタンをタップする
3. iPhoneのFiles appからCSVファイルを選択する
4. インポート処理が実行される
5. 結果（取込件数・新規登録メニュー数・スキップ数）が表示される

**代替フロー：**
- 重複セッション → スキップ（既存データは上書きされない）
- アプリ未登録のメニュー → 自動で新規登録してから取り込む
- session_idが空欄 → 同日同メニューを同一セッションとして取り込む
- ファイル形式が不正 → エラーメッセージを表示

## テスト観点メモ（B-09・B-10向け）

将来の単体テスト・E2Eテスト構築時の参考として、各UCの主要テスト観点を記載する。

### 単体テスト観点（B-09）

| 対象関数 | テスト観点 |
|---|---|
| `orm(w, r)` | 正常値・境界値（r=0, w=0）・小数点精度 |
| `onNameInput()` | 全角30文字ちょうど・31文字目でカット・サロゲートペア |
| `parseCSV()` | 引用符あり・なし・空欄・改行コード差異 |
| `importCSV()` | 重複セッション・未登録メニュー・session_id空欄 |
| `menuStats()` | セッションなし・筋トレ・有酸素の分岐 |
| `buildStrengthRows()` | 1RM・ボリュームの計算値がCSVに正しく出力されるか |

### E2Eテスト観点（B-10）

| UC-ID | E2E観点 |
|---|---|
| UC-01 | メニュー追加→一覧に表示される |
| UC-06 | セット記録→メニュー詳細の統計値が更新される |
| UC-03/04 | アーカイブ→一覧から消える→復元→再表示される |
| UC-05 | 完全削除→メニューと記録が消える |
| UC-16〜18 | カレンダーで記録日が強調→タップ→セッション遷移 |
| UC-21〜23 | エクスポート→ファイルダウンロード・インポート→記録が反映 |
